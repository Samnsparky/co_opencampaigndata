co_opencampaigndata working API spec

Easy programmatic access to Colorado's campaign finance data.
Just sign up for an API key (link coming soon) and make GET requests at the endpoints below. All parameters except apiKey are optional.
GET /expenditures?apiKey={apiKey}
URL
/expenditures
Methods
GET
Alternatives
/expenditures.json (default), /expenditures.json
Description
Retrieve reports for political committees.
Allowed parameters
{?offset}
The number of records to skip in the dataset.
{?limit}
The max number of records to return.
{?commiteeID}
The id of the committee to filter for.
{?minAmount}
Only include expenditures above or equal to this amount in US dollars.
{?maxAmount}
Only include expenditures below or equal to this amount in US dollars.
{?startDate}
Only include expenditures occurring on or after this date. Use ISO 8601 string.
{?endDate}
Only include expenditures occurring on or before this date. Use ISO 8601 string.
{?lastName}
Only include payments made to individuals with this last name or organizations with this name
{?firstName}
Only include payments made to individuals with this first name.
{?middleInitial}
Only include payments made to individuals with this middle initial.
{?address}
Only include expenses paid to entities at this address.
{?city}
Only include expenses paid to entities in this city.
{?state:2}
Only include expenses paid to entities in this state. Use abbreviation like CO.
{?zip:2}
Only include expenses paid to entities in this zip code.
{?recordID}
The ID of the record to return. This ID is unique within the dataset.
{?startFiledDate}
Only include expenditures filed on or after this date. Use ISO 8601 string.
{?endFiledDate}
Only include expenditures filed on or before this date. Use ISO 8601 string.
{?expenditureType}
Only include expenditures with the given string expenditure type.
{?paymentType}
Only include expenditure reports with the given string receipt type.
{?disbursementType}
Only include reports with the given string disbursement type.
{?electioneering}
If 1, only include expenditures marked as electioneering. If 0, only include expenditures not marked as electioneering. Use 0 or 1 to avoid localization.
{?committeeType}
Only include expenses from committees with the given string type.
{?committeeName}
Only include expenses from the committee with this name. Case insensitive.
{?candidateName}
Only include expenses from candidates with the given name. Case insensitive.
{?employer}
Only include expenses paid to those with the given employer.
{?occupation}
Only include expenses paid to those with the given occupation.
{?amended}
If 1, only include records that were amended. If 0, only include records that are amendments. Use 0 or 1 to avoid localization requirements.
{?amendment}
If 1, only include records that are amendments. If 0, only include records that were not amendments. Use 0 or 1 to avoid localization requirements.
{?amendedRecordID}
Only include records that amended the record with this recordID.
{?jurisdiction}
Only include expenses reported as falling under the given jurisdiction.
{?fields}
Comma separated list of record fields to include in the response JSON. See the example response for a listing of available fields.

/contributions
Methods
GET
Alternatives
/contributions.json (default), /contributions.json

Allowed parameters

{?middleInitial}

{?city}
Only include contributions originating from this city.

GET /loans?apiKey={apiKey}
URL
/loans?apiKey={apiKey}
Methods
GET
Alternatives
/loans.json (default), /loans.json
Description
Retrieve reports of reports of loan payment activity.
Allowed parameters
{?limit}
The max number of records to return.
{?offset}
The number of records to skip in the dataset.
{?commiteeID}
The id of the committee to filter for.
{?minPaymentAmount}
Only include loan activity involving payment amounts above or equal to this threshold in US dollars.
{?maxPaymentAmount}
Only include loan activity involving payment amounts below or equal to this amount in US dollars.
{?startDate}
Only include payments occurring on or after this date. Use ISO 8601 string.
{?endDate}
Only include payments occurring on or before this date. Use ISO 8601 string.
{?name}
Only include payments from individuals or organizations with this name.
{?address}
Only include payments with contributors from this address.
{?city}
Only include payments originating from this city.
{?state:2}
Only include payments originating from this state. Use abbreviation like CO.
{?zip:5}
Only include payments from this five digit zip code.
{?recordID}
The ID of the record to return. This ID is unique within the dataset.
{?startFiledDate}
Only include payments reported after this date. Use ISO 8601 string.
{?endFiledDate}
Only include payments recorded before this date. Use ISO 8601 string.
{?loanType}
Only include payments with the given string contribution type.
{?loanSourceType}
Only include activity on loans from this type (possible values include "business" and "candidate" case insensitive).
{?minLoanAmount}
Only include reports with loans greater than or equal to the given loan amount (in USD).
{?maxLoanAmount}
Only include reports with loans greater than or equal to the given loan amount (in USD).
{?minLoanStartDate}
Only include loans starting on or after this date. Use ISO 8601 string.
{?maxLoanStartDate}
Only include loans start on or before this date. Use ISO 8601 string.
{?committeeType}
Only include loans for the given string committee type.
{?committeeName}
Only include loans for committees with the given name.
{?minInterestRate}
Only include loans with an interest greater than or equal to this interest rate. This value should be less than or equal 1 and greater than or equal to 0.
{?amended}
If 1, only include records that were amended. If 0, only include records that are amendments. Use 0 or 1 to avoid localization requirements.
{?amendment}
If 1, only include records that are amendments. If 0, only include records that were not amendments. Use 0 or 1 to avoid localization requirements.
{?amendedRecordID}
Only include records that amended the record with this recordID.
{?jurisdiction}
Only include loan activity reported as falling under the given jurisdiction.
{?loanBalance}
Only include loan activity reported as falling under the given jurisdiction.
{?fields}
Comma separated list of record fields to include in the response JSON. See the example response for a listing of available fields.
